# 班级人脸识别考勤系统 (BS架构)

## 核心技术栈与架构
- **前端**：Vue.js 2 + Element UI + ECharts，纯静态单页面应用。
- **后端**：Python Flask，提供 RESTful API 支持。
- **视觉引擎**：**InsightFace** (`buffalo_l` 模型) 搭配 **ONNXRuntime**，解决 Windows 平台下的 TensorFlow GPU 兼容与性能瓶颈；**OpenCV** 处理图像矩阵。

## 系统核心功能

### 1. 基础考勤与活体检测
- **流畅采集**：前端基于 HTML5 MediaDevices 调用摄像头，Base64 实时回传人脸特征。
- **高能效引擎**：通过 InsightFace 高精度模型执行特征嵌入(Embedding)、坐标边界(BBox)的本地解算。
- **本地静态活体筛查**：不再依赖外部云 API，通过单帧特征、防屏拍与基础分析进行被动活体检测（Passive Liveness）。
- **记录筛选**：支持按日期筛选历史打卡轨迹，统一留痕。

### 2. 角色隔离与权限 (RBAC)
- **学生模式**：极简界面，仅保留实时考勤打卡功能，自动隐藏全班人脸库库、敏感统计报表及控制台，保护隐私。
- **教师模式**：拥有最高管理权限，可执行批量发图识别、新增人员录入（支持本地上传/摄像头直采）、统计报表查询与多表联动等管理操作。

### 3. 合照多目标识别与活动统计
- **批量识别**：一键上传 10-50 人大合照，后端执行动态大尺度下采样，加速定位并进行库内特征 Top-N 向量距离比对。
- **结果绑定**：自动统计识别总人数、成功匹配数与未知身份数，生成对应名单列表。
- **可视化报表**：无缝对接 ECharts，通过自适应挂载点实时响应渲染各个学生参与活动的频次排名柱状图。

### 4. 情绪分析
- **情绪提取**：在进行考勤（单人）或合照识别（多人）时同步解析面部情绪（平静、开心、愤怒、专注等）。
- **聚类统计**：前端动态生成情绪分类统计饼形图，直观展示一段时间内班级总体情感状态分布。

## 环境搭建与运行指南

### 1. 依赖安装
系统需要 Python 3.8+ 环境，进入 `backend` 目录，安装对应依赖：
```bash
cd backend
pip install -r requirements.txt
```
*(主要包含 Flask, opencv-python, onnxruntime, insightface 组件)*

### 2. 模型下载与加载 (InsightFace)
`app.py` 中 `FaceAnalysis(name="buffalo_l")` 首次初始化时会自动拉取模型。请保持网络通畅。
如需离线部署，请提前将 `buffalo_l.zip` 解压至 `~/.insightface/models/buffalo_l/` 目录下。

### 3. 运行系统
在终端以主进程模式启动 Flask 路由服务：
```bash
cd backend
python app.py
```
> 目前 Backend `app.py` 为避免在生成快照 `static/snapshots` 时的热重载死循环，已关闭 Flask Debug 功能。

启动完毕后，在浏览器直接开打前端 HTML 入口即可：
```text
frontend/index.html
```

## 使用场景指南

1. **导入人脸库 (教师模式)**：
   - 使用 【批量上传】，遵循 `学号-姓名-专业-性别.jpg` 命名规范（如：`2026001-张三-网安-男.jpg`）。
   - 或者手动调取摄像头当面添加。
2. **实时考勤**：在【实时考勤】内开启浏览器摄像头，调整姿势在光线明显且单人出镜时点击【考勤打卡】。
3. **活动合照 (教师模式)**：适合会议签到与大班实验课，添加活动名称并传入大图，静待批量特征反馈即可。
4. **统计分析**：【统计报表】页内可查阅出勤人数、情绪占比、合照热度排行以及下发导出 CSV 文档操作。
